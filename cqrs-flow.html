Domain — Book Entity
İş Kuralı Entity'de Uygulanır

// Book.cs (Domain Entity) public void DecreaseCopy() { if (AvailableCopies <= 0) throw new DomainException( $"'{Title}' kitabının müsait kopyası kalmadı."); AvailableCopies--; // private set — sadece entity değiştirebilir }

4
Application — GetOverdueLoansQueryHandler
QueryHandler Sadece Okur
SaveChangesAsync yok. Entity metodu yok. Hiçbir şeyi değiştirmiyor. Repository'den okur, DTO'ya dönüştürür.
// GetOverdueLoansQueryHandler.cs public async Task<List<LoanResponseDto>> HandleAsync( GetOverdueLoansQuery query, CancellationToken ct) { var loans = await _loanRepository .GetOverdueLoansAsync(ct); // sadece okuma return loans.Select(l => new LoanResponseDto { Id = l.Id, BookTitle = l.Book.Title, // ... }).ToList(); }
▸ koda bak
5
Infrastructure — LoanRepository
Repository Filtrelenmiş Sorgu Çalıştırır
EF Core'un Where + Include ile optimize sorgusu. Sadece geciken kayıtlar çekilir. Book ve Member ilişkileri JOIN ile tek sorguda gelir.
// LoanRepository.cs public async Task<List<Loan>> GetOverdueLoansAsync(CancellationToken ct) { var today = DateTime.UtcNow; return await _context.Loans .Include(l => l.Book) // JOIN ile Book verisi .Include(l => l.Member) // JOIN ile Member verisi .Where(l => l.ReturnDate == null && l.DueDate < today) // filtre .ToListAsync(ct); }
